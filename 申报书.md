# 2026 MoonBit 国产基础软件开源大赛 项目申报书

## 1. 项目名称
**moonbit-toml** (MoonBit TOML Data Engineering Toolkit)

---

## 2. 项目简介
`moonbit-toml` 是一款基于 **MoonBit** 语言开发的**生产级 TOML 数据工程工具包**。

目前 MoonBit 生态已有基础的 TOML 解析实现（如 `bobzhang/toml`），但仅停留在“读取文件”层面。本项目**定位为应用级工具包**，专注于解决开发者在实际工程中的**高阶痛点**：如何合并配置、对比差异、跨格式迁移数据、动态构建复杂结构以及进行 Schema 校验。

本项目不仅提供标准的 TOML v1.0.0 解析功能，还创新性地引入了 **Builder 构建模式**、**配置合并/差异分析 (Merge/Diff)**、**多格式互操作 (JSON/XML/SQL/CSV)** 以及**结构统计 (Statistics)** 等企业级特性，旨在成为 MoonBit 生态中处理配置数据的**一站式解决方案**。

---

## 3. 项目方向与适用场景

### 项目方向
- **原创生态库**：MoonBit 基础工具链/配置管理库

### 适用场景
1.  **微服务与后端开发**：用于解析服务配置文件（如 `config.toml`），提供**配置热重载**与**多环境配置合并 (Merge)** 功能。
2.  **数据迁移与互操作**：在 MoonBit 生态中充当数据交换桥梁，实现 TOML 与 JSON、YAML、CSV、SQL DDL 之间的无缝映射，辅助旧系统迁移。
3.  **DevOps 与 CI/CD**：自动化脚本中的**配置差异分析 (Diff)**、**版本对比**、**模板生成**。
4.  **工具链开发**：作为构建系统或 CLI 工具的底层配置引擎，提供强类型访问与路径查询 (Query)。

## 5. 原创性与具体差异化说明

- **原创声明**：本项目为**原创生态库**。
- **创作模式**：由创作人 **郭亚斌** 主导架构设计与核心逻辑实现，辅以 **AI** 进行代码生成、语法校验、重构建议与测试脚本编写 (个人 + AI 创作)。
- **与现有项目 (`bobzhang/toml`) 的具体区别**：
  
  虽然社区已有 `bobzhang/toml` 项目，但本项目在**功能深度**与**工程应用**上具有明确的互补性和不可替代性：

  | 功能维度 | 现有项目 (`bobzhang/toml`) | 本项目 (`moonbit-toml`) | 具体用户价值 |
  | :--- | :--- | :--- | :--- |
  | **核心定位** | **语法解析 (Parsing)** | **数据治理 (Manipulation)** | 从“能读取”进阶到“好管理”。 |
  | **配置合并** | ❌ **不支持**<br>(用户需手写递归逻辑) | ✅ **支持 (`merge`)**<br>自动处理 Base/Overlay 深度合并 | 解决微服务多环境配置叠加痛点，减少 90% 样板代码。 |
  | **变更检测** | ❌ **仅支持全量比较** (`==`) | ✅ **支持 (`diff`)**<br>输出细粒度变更日志 | 支持配置热更新时的 Diff 审计，精准定位 `Added/Modified` 节点。 |
  | **格式互转** | ❌ **仅输出 TOML 格式** | ✅ **多格式导出**<br>支持 JSON, SQL DDL, CSV, XML | 助力跨语言迁移，如直接将 TOML 转为 SQL `CREATE TABLE` 语句。 |
  | **动态构建** | ⚠️ **依赖底层 Map 操作**<br>(类型不安全，易出错) | ✅ **Builder API**<br>类型安全的链式构建接口 (`builder.set_*`) | 简化复杂配置结构的代码生成，提供 IDE 友好体验。 |
  | **业务校验** | ❌ **仅保证语法合法** | ✅ **Schema 校验**<br>支持定义字段类型约束 | 拦截“端口为负数”等语法正确但业务非法的配置。 |
  | **路径查询** | ❌ **需用户手动逐层解包**<br>(易 panic) | ✅ **路径解析**<br>支持 `resolve_path("server.host")` 安全访问 | 简化深层嵌套数据的读取，避免繁琐的空指针检查。 |

- **技术独立性**：项目**未移植**自任何现有的 C/Rust/Go 实现，而是从零开始基于 MoonBit 的**代数数据类型 (ADT)**、**模式匹配**及**内部可变性 (Interior Mutability)** 机制原生构建。
- **开源协议**：采用 **Apache-2.0** 许可证，开源协议友好。
