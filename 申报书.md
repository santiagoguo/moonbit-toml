## 一、项目背景与意义

TOML (Tom's Obvious, Minimal Language) 作为一种广泛使用的配置文件格式，已成为 Rust (`Cargo.toml`)、Python (`pyproject.toml`) 等现代语言生态的事实标准。然而，MoonBit 语言生态中目前仍缺乏一个功能完整、经过严格测试的 TOML 处理库。

本项目 `moonbit-toml` 旨在填补这一空白，提供一套**从底层解析到高级数据操作**的完整解决方案。不仅实现了 TOML v1.0.0 规范的核心语法，更创新性地引入了配置合并、差异计算、路径查询、格式转换（JSON/XML/YAML/CSV/SQL）、Schema 校验等高级特性，使其成为 MoonBit 生态中配置处理领域的标杆级基础设施。

## 二、技术方案

### 2.1 核心架构设计

本项目彻底摒弃了传统的 `mut self` 面向对象模式，全面采用符合 MoonBit 语言特性的**函数式状态传递架构**：

```
┌─────────────────────────────────────────────────────┐
│                   moonbit-toml                       │
├─────────────────────────────────────────────────────┤
│  📦 核心层 (Core Layer)                              │
│  parse(input) → TomlValue? / serialize(value) → String│
│  ┌───────────┐  ┌──────────┐  ┌───────────┐         │
│  │ Lexer     │→ │ Parser   │→ │ AST Builder│        │
│  │ (词法分析) │  │ (语法树) │  │ (状态传递) │         │
│  └───────────┘  └──────────┘  └───────────┘         │
├─────────────────────────────────────────────────────┤
│  🛠️ 扩展层 (Extension Layer)                         │
│  Query API │ Merge/Diff │ Builder │ Validation       │
│  Stats     │ Transform  │ Linter  │ Extended Formats │
├─────────────────────────────────────────────────────┤
│  🔄 转换层 (Serialization Layer)                     │
│  JSON │ XML │ YAML │ CSV │ SQL │ Properties │ JSON5 │
└─────────────────────────────────────────────────────┘
```

### 2.3 技术亮点与创新

1. **纯函数式架构**：针对 MoonBit 不支持 `mut self` 的特性，采用 `(State, Result) = fn(State)` 的状态传递模式与元组返回机制，彻底规避了副作用，代码具备极高的可测试性与可推理性。
2. **内部可变性最佳实践**：对 `Array`/`Map` 集合类型严格使用内置的内部可变性（Interior Mutability），避免 `let mut` 带来的编译器警告，符合 MoonBit 语言设计哲学。
3. **零外部依赖**：核心解析逻辑完全基于 MoonBit 标准库实现，无第三方依赖，编译体积轻量，启动速度极快。
4. **强大的类型系统利用**：充分利用 MoonBit 的代数数据类型 (`enum`) 与模式匹配 (`match`)，在编译期拦截非法状态，运行时零 `panic`（除致命语法错误外）。
5. **生产级 API 设计**：提供 Builder 模式、链式查询、Schema 校验、多格式导出等企业级特性，可直接用于微服务配置中心、CLI 工具、数据管道等复杂场景。
